## How to Update This Changelog

### For Maintainers

When merging a PR, update the `[Unreleased]` section with the changes:

1. **Added** - for new features
2. **Changed** - for changes in existing functionality
3. **Deprecated** - for soon-to-be removed features
4. **Removed** - for now removed features
5. **Fixed** - for any bug fixes
6. **Security** - in case of vulnerabilities

### For Contributors

You don't need to update this file directly. Maintainers will update it when merging your PR.

### Release Process

When creating a new release:

1. Move items from `[Unreleased]` to a new version section
2. Add the release date
3. Create a new empty `[Unreleased]` section
4. Update the version links at the bottom

Example:
```markdown
## [1.1.0] - 2025-02-01

### Added
- New feature X
- New feature Y

### Fixed
- Bug fix Z
```
